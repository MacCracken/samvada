# Contributing to samvada

Thank you for your interest in contributing to samvada — the
Cyrius dbus client for the AGNOS suite.

## Prerequisites

- [Cyrius](https://github.com/MacCracken/cyrius) — version
  pinned in `cyrius.cyml [package].cyrius` (currently 5.7.48).
  Ships `cc5`, the `cyrius` CLI, and the stdlib samvada depends on.
- `gcc` / `clang` (for the libsystemd C shim only —
  `deps/samvada_main.c` is built by consumers, not by samvada
  itself, but CI compile-checks it).
- `libsystemd-dev` (Debian/Ubuntu) / `systemd-devel` (Fedora) /
  `systemd` (Arch — already installed) — required only when
  compile-checking `deps/samvada_main.c` locally. samvada's
  Cyrius bundle does not link libsystemd; consumers do.

## Development Workflow

1. Fork and clone the repository.
2. Resolve stdlib deps: `cyrius deps` (populates `lib/`).
3. Build the smoke binary: `cyrius build src/main.cyr build/samvada`.
4. Run the test suite: `cyrius test`.
5. Make changes under `src/`, `tests/`, `deps/`, or `docs/`.
6. Re-run lint / fmt / vet locally before opening a PR.
7. Submit a PR.

## Project Structure

```
src/main.cyr           Standalone smoke entry point ("cyrius build" path).
src/lib.cyr            Include chain — single source of truth for [lib].
src/samvada.cyr        Public API surface (stable across v0.x).
src/samvada_ffi.cyr    FFI fn-table layout (slot offsets + helpers).
src/test.cyr           Top-level test entry referenced by [build].test.
tests/samvada.tcyr     Primary CPU test suite.
tests/samvada.bcyr     Benchmark harness.
tests/samvada.fcyr     Fuzz harness.
deps/samvada_main.c    libsystemd C shim — built by consumers.
dist/samvada.cyr       Generated by `cyrius distlib` — checked in.
docs/                  ADRs, architecture, guides, examples, dev state.
cyrius.cyml            Package manifest (toolchain pin, [lib], [deps]).
```

## Running the Local Gate Sweep

```sh
cyrius deps                                    # resolve stdlib
cyrius lint src/*.cyr tests/*.tcyr             # warnings -> failures
cyrius fmt --check src/*.cyr tests/*.tcyr      # no drift
cyrius vet src/main.cyr                        # static checks
cyrius distlib                                 # dist/samvada.cyr stays fresh
cyrius build src/main.cyr build/samvada        # smoke build
cyrius test                                    # 32 asserts as of v0.2.1
cyrius bench tests/samvada.bcyr                # 4 CPU baselines
# C shim compile-check (consumer-side build):
cc -Wall -Wextra -Werror -c deps/samvada_main.c \
   $(pkg-config --cflags libsystemd) -o /tmp/samvada_main.o
```

CI runs the same sweep on every push. Drift on any gate fails
the build.

## Adding a Module

1. Create `src/mymodule.cyr` with a `# @public` or `# @internal`
   marker on line 1 and a comment block describing its surface.
2. Add `include "src/mymodule.cyr"` to `src/lib.cyr` in
   dependency order.
3. Add `"src/mymodule.cyr"` to `[lib].modules` in `cyrius.cyml`.
4. Add CPU asserts to `tests/samvada.tcyr`.
5. Regenerate the bundle: `cyrius distlib` (CI diff-checks this).
6. Update `CHANGELOG.md` under `## [Unreleased]`.
7. Update `docs/development/state.md` if the change is
   architectural (new modules, new public fns, version bump).

## Adding an FFI Slot

When a new logind / dbus call needs to flow through samvada:

1. Append a slot offset fn in `src/samvada_ffi.cyr` AT THE END
   (after `samvada_slot_kind`). The append-after-kind invariant
   means `samvada_slot_kind` stays at offset 64 forever — never
   reorder or insert before it.
2. Bump `samvada_ffi_size()` by 8 per added slot.
3. Add the matching C wrapper to `deps/samvada_main.c`. Keep
   wrappers ≤6 args so dispatch stays `fncall6`-safe on x86_64
   SysV + aarch64.
4. Add a slot-offset CPU assert to `tests/samvada.tcyr` so
   reordering is caught at test time.
5. Add a public Cyrius wrapper in `src/samvada.cyr` (dispatch
   via `samvada_ffi_get_slot` + `fncallN`).

## Code Style

- Use `#` comments (not `//`).
- Prefix module-internal functions with `_`.
- Use `alloc(N)` for heap structs, `store64` / `load64` for
  field access — Cyrius has no struct types, the offset-table-
  on-heap pattern is canonical.
- Errors are negative sd-bus errnos for FFI-touching paths so
  the C convention passes through unchanged.
- `cyrius fmt` is the arbiter of formatting; CI fails on drift.
- Reach for `println(s)` (single-arg) and `print(s, len)`
  (two-arg) — there is no `print(s)` single-arg form.

## Commit Messages

Follow [Keep a Changelog](https://keepachangelog.com/)
categories. Common samvada prefixes:

- `feat(api):` / `feat(ffi):` / `feat(shim):` — new functionality
- `fix:` — bug fix
- `docs:` — docs-only change
- `chore(release):` — VERSION + CHANGELOG bump
- `chore(m1):` / `chore(m2):` etc. — milestone closeout
